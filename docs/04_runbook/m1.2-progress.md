## W1.2.7 产出

- `scripts/codegen/generate-ts-schemas.mjs` 已从 schema-name stub 升级为真实 `json-schema-to-typescript` 编译器。
- `scripts/codegen/format-json-schemas.mjs` 显式格式化 Pydantic JSON Schema 产物，避免 TS generator 隐式修改输入。
- `packages/schemas-ts/src/generated/*.ts` 现在导出对应 Pydantic 模型的 TypeScript 类型，而不是 `*_SCHEMA_NAME` 占位常量。
- `packages/schemas-ts/src/generated/index.ts` 作为 generated barrel，使用 `.js` ESM specifier；`packages/schemas-ts/src/index.ts` 通过 `export type *` 对外暴露 90 个生成类型。
- `json-schema-to-typescript` 作为 workspace 根 devDependency，仅用于 codegen。

## W1.2.8 产出

- `packages/schemas/tests/test_w1_2_8_error_code_coverage.py` 维护当前 Pydantic schema 层可判定的 spec 错误码集合，并做三重守卫：集合内 code 必须出现在 spec 文本中、实现层 custom code 不得超出该集合、集合内 code 必须在行为测试中通过 `_assert_validation_error_contains(...)` 断言。
- 补齐 WorkflowGraph / NodeContract / ContextPack / EvidencePack / EvaluationResult / RepairPatch 的缺口测试，覆盖 `WG_L2_EVAL_MISSING_TARGET`、`NC_L2_EVAL_NO_CRITERIA`、`NC_L2_REPAIR_NO_STRATEGIES`、`ER_BUILD_CRITERIA_MISMATCH`、`RP_BUILD_EMPTY_OPERATIONS` 等。
- 对会被 Pydantic 字段约束提前截获的空列表规则做最小调整：保留 JSON Schema 的 `minItems`，将运行时错误映射交给模型 invariant，确保返回 spec 错误码。
- 非 spec custom code 已从 schema 层收敛：非 WorkflowGraph 的 `schema_version` 改为 `Literal["0.1.0"]` 原生校验，duplicate/id/human/memory 等未在 schema spec 错误码表锁定的规则只抛普通 `ValidationError`。
- 当前 schema 层 spec custom error code 覆盖：WG 13、NC 6、CP 2、EP 3、ER 4、RP 4、SE 3，共 35 个。L1 JSON 解析、L3/L4 Compiler/外部注册表、runtime/apply/persist/SSE 状态型错误不在 `packages/schemas` 中强行实现。
