## W1.5.1 范围

本切片启动 M1.5 桌面 Shell 工作，但不新增 production dependency，不接入 Electron Forge/Vite/React/Tailwind 打包链；先把 ADR-0006 与 `http_sse.md §7` 已锁定的安全和通讯边界落成可测试 TypeScript 模块：

- `@cw/desktop` 的 `build` / `typecheck` / `test` / `lint` 从占位 echo 收敛为真实 TypeScript gate；`test` 使用 Node 内置 `node:test`，不新增 Vitest 依赖。
- 新增 strict `apps/desktop/tsconfig.json`，启用 `strict`、`noUncheckedIndexedAccess`、`exactOptionalPropertyTypes`，输出到已忽略的 `apps/desktop/dist/`。
- `src/main/runtime.ts` 只接受 sidecar stdout `READY <port>`，将端口收敛到 `http://127.0.0.1:<port>/cw/v1`；拒绝 0、65536、非整数、URL 形 READY；`buildRuntimeConnectionInfo()` 在把 connection info 暴露给 preload 前校验 token 非空且无空白/控制字符。
- `src/main/security.ts` 固定 BrowserWindow 安全默认值：`contextIsolation=true`、`sandbox=true`、`nodeIntegration=false`、`webSecurity=true`、`allowRunningInsecureContent=false`；CSP 默认只允许 `connect-src 'self' http://127.0.0.1:*`，拒绝 `unsafe-eval`、`unsafe-inline`、`localhost`、`0.0.0.0` 和通配 HTTP 源；开发 websocket 仅允许显式 opt-in 的 `ws://127.0.0.1:*`。
- `src/preload/contract.ts` 定义 `window.cw.runtime` 的最小 preload contract，要求 renderer 只传相对 API path；preload 注入 `Authorization: Bearer <token>`、`X-Cw-Client: electron-renderer`、可选 `X-Project-Id` 与 `Idempotency-Key`，并拒绝 renderer 覆盖保留 header 或注入 CR/LF。

边界说明：W1.5.1 不创建真实 Electron `BrowserWindow`、不实现 `child_process.spawn`、不启动 Vite renderer、不实现 React Flow Canvas/Chat/Drawer、不修改 accepted spec/ADR、不提升 M1.5 完成度到 UI 可运行。Electron Forge、React 18、Tailwind、React Flow、electron-updater 等 production dependency 接入需单独切片并按 AGENTS 先给出版本与理由。

## W1.5.2 范围

本切片继续推进 M1.5 桌面主进程基础，但仍不新增 production dependency、不创建真实 Electron `BrowserWindow`、不实现 IPC 或 renderer；只把 `http_sse.md §7` 与 ADR-0006 已锁定的 sidecar spawn/token/READY 协作协议收敛为可测试 TypeScript 模块：

- 新增 `src/main/sidecar.ts`，导出 `generateRuntimeAuthToken()`、`buildRuntimeSidecarArgs()`、`buildRuntimeSidecarEnvironment()` 与 `startRuntimeSidecar()`。
- `generateRuntimeAuthToken()` 默认生成 16 random bytes 的 base64 token，对齐 `http_sse.md §1.2 / §7` 的 128-bit ephemeral bearer token 要求；token 仅作为内存值进入子进程环境，不写磁盘。
- `buildRuntimeSidecarArgs()` 强制桌面主进程拥有 `--http-port=0`，拒绝调用方传入 `--http-port` 或 `--http-port=<n>`，避免 renderer/外部配置把 sidecar 暴露到固定端口。
- `buildRuntimeSidecarEnvironment()` 注入 `CW_RUNTIME_AUTH_TOKEN`，复用 `normalizeRuntimeAuthToken()` 拒绝空白/控制字符 token，且不 mutate 调用方传入的 base env。
- `startRuntimeSidecar()` 使用 Node `child_process.spawn`（`windowsHide=true`、`stdio=["ignore","pipe","pipe"]`），等待 stdout 中的 `READY <port>` 并返回 `RuntimeSidecarSession` `{ process, ready, connection, stop() }`；提前 `error` / `exit`、缺 stdout pipe、READY timeout 都以结构化错误 fail closed，timeout 会 `SIGTERM` 清理子进程。
- 测试覆盖 128-bit token、port 参数所有权、env 注入不可变、READY 前日志容忍、connection info 输出、unsafe token factory 不 spawn、READY timeout kill、READY 前退出失败。

边界说明：W1.5.2 仍不实现 runtime.lock 复用、真实 packaged runtime path 发现、Electron IPC、renderer `window.cw.runtime.connectionInfo()` 实装、EventSource/fetch 客户端或跨平台安装包；这些需要后续 Electron/packaging 切片或 runtime harness lock 切片继续推进。

- 新增 `acquireRuntimeLock()`，先创建 `.agent-workflow/locks/`，再用 Node `writeFile(..., flag: "wx")` 做 exclusive-create 骨架；missing lock 直接 acquire，active lock 按 retry/timeout 等待，corrupt/unreadable lock fail closed。
- stale lock 先通过现有 `inspectRuntimeLock()` 判定，再进入同一套 desktop helper 使用的短生命周期 mutation guard，在 guard 内重新 inspect 后才删除并重新 acquire；release 也在同一 guard 内复查当前文件内容仍等于本 lease 写入内容后才删除，避免同一 helper 之间误删已经被替换的 lock。
- `src/main/index.ts` 导出 acquire/release 相关常量、函数和类型，供后续真实主进程 orchestration 复用。
