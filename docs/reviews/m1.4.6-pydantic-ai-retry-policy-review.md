# W1.4.6 Pydantic AI Retry Policy Review

| Item        | Value                                                       |
| ----------- | ----------------------------------------------------------- |

| Verdict     | PASS                                                        |

## Scope

Reviewed the W1.4.6 narrow slice against `specs/protocols/agent_adapter.md`, `specs/schemas/node_contract.md`, ADR-0002, and `docs/04_runbook/m1.4-progress.md`.

The review focused on:

- Whether the default SDK session projects SDK-supported retry dimensions into `Agent(retries=...)`.
- Whether `output_validation_retries` maps to `retries["output"]`.
- Whether integer `tool_retries` maps to `retries["tools"]`.
- Whether per-tool retry maps fail closed because Pydantic AI `AgentRetries` cannot express them.
- Whether explicit nonzero `model_retries` fail closed because Pydantic AI 1.107.0 `AgentRetries` only supports `tools` and `output`.
- Whether unsupported retry responsibilities remain outside W1.4.6: `max_attempts`, model HTTP retry utilities, backoff, escalation, timeout/cancel, cost, human/deferred resume, and real registries.

## First Review

Verdict: FAIL.

Finding:

| ID  | Severity | Description                                                                                                                                   | Resolution                                                                                                                                 |
| --- | -------- | --------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| F1  | high     | `model_retries` was accepted into the adapter projection but could not be enforced by Pydantic AI `AgentRetries`, so it was dropped silently. | `_sdk_agent_retries()` now rejects explicit nonzero `model_retries` with `AA_PREPARE_INCOMPATIBLE_ADAPTER`; regression coverage was added. |

## Final Review

Verdict: PASS.

The reviewer confirmed:

- `PydanticAIRunRequest` carries a serializable `PydanticAIRetryPolicy` projection.
- Default retry projection passes `{"tools": 2, "output": 2}` to the SDK agent without treating schema-default `model_retries=2` as supported.
- Explicit nonzero `model_retries` fails closed with `AA_PREPARE_INCOMPATIBLE_ADAPTER`.
- Per-tool retry maps fail closed with `AA_PREPARE_INCOMPATIBLE_ADAPTER`.
- SDK mapping remains scoped to the Pydantic AI 1.107.0 `AgentRetries` shape: `tools` and `output`.
- No production dependency changes were introduced.
- ADR-0002 holds: Pydantic AI remains a lazy import inside the adapter boundary.
