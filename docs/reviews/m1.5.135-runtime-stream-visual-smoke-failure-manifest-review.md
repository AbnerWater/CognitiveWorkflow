# M1.5.135 Runtime Stream Visual Smoke Failure Manifest Review

## Scope

W1.5.135 hardens the dev-only Runtime Stream visual smoke matrix failure path. The runner must preserve `matrix.json` when a case fails, cannot leak raw child stdout/stderr or raw JSON parse messages, and must reject invalid or non-object case evidence before the matrix can pass.

Files reviewed:

- `apps/desktop/scripts/runtime-workbench-visual-smoke-matrix.cjs`
- `docs/04_runbook/m1.5-progress.md`

Out of scope:

- Production runtime/session/store/React shell behavior.
- Accepted spec / ADR / schema / generated schema changes.
- Dependency, runtime API, IPC / `window.cw`, renderer storage, React Flow, Tailwind, packaging/updater changes.
- Raw payload, `rawData`, metadata object/key/value display.

## Independent Review

Reviewer: Carver (`019ef9ab-0fc0-7872-9bbc-5afb053febd0`)

Initial result: CONDITIONAL_PASS.

Findings:

- P1: unreadable or malformed JSON failure used raw `error.message`, which can include an excerpt of malformed JSON content and leak it into `matrix.json` or CI logs.
- P2: valid JSON literal `null` could false-pass when the child exited 0 and wrote no stderr, because `null` was also the unparsed sentinel.

Open questions:

- Confirm whether raw child stdout/stderr must be suppressed in terminal output as well as manifest output.
- Include child close `signal` in the process summary so non-exit termination is auditable.

Second review result: NOT PASS.

Additional findings:

- P2: the first fix still allowed valid JSON `null` to false-pass because root validation was gated by `result !== null`.
- P3: a stale untracked `apps/desktop/.agent-workflow/locks/runtime.lock` existed after local Electron startup and needed cleanup or explicit exclusion before closure.

Resolution:

- JSON read/parse failures now use safe summaries only: unreadable files record fs `error.code`, syntax errors record `case JSON was not valid JSON`, and raw `error.message` is not referenced.
- A `parsedJson` sentinel separates parse success from the parsed value, so valid JSON `null`, arrays, and primitives fail with `case JSON root was not an object`.
- Child stdout/stderr are captured for lengths only and are no longer mirrored to parent stdio.
- Manifest `process` summary records only `exitCode`, `signal`, `errorCode`, `stdoutLength`, and `stderrLength`.
- Child spawn `error` resolves through the normal manifest path with a safe `errorCode` summary instead of rejecting before `matrix.json` can be written.
- The stale runtime lock was confirmed dead by PID check and removed from the working tree.

Final result: PASS, no blocker/P1/P2/P3 findings.

Carver confirmed the final diff has no `error.message`, no `process.stdout.write`, no `process.stderr.write`, preserves `matrix.json` before throwing on matrix failures, and does not introduce production/spec/generated/dependency/API/IPC/window.cw/storage/React Flow/Tailwind/packaging/updater drift.
