## Scope

Reviewed the W1.4.3 narrow slice against `specs/protocols/agent_adapter.md`, `specs/schemas/stream_event.md`, ADR-0002, and `docs/04_runbook/m1.4-progress.md`.

The review focused on:

- Whether `capabilities().streaming=True` is now truthful without opening Toolset/MCP/HITL/deferred/cancel/multimodal/evidence capabilities.
- Whether Pydantic AI remains an optional lazy import inside the adapter boundary.
- Whether `agent.run_stream_events(...)` is consumed through the `AgentEventStream` async context manager.
- Whether SDK stream events are translated only to spec-defined StreamEvent types.
- Whether tests cover the stream bridge and the first review's failure cases.

## First Review

Verdict: FAIL.

Findings:

| ID  | Severity | Description                                                                                                                                           | Resolution                                                                                                                               |
| --- | -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| F1  | high     | `RetryPromptPart` from `FunctionToolResultEvent` was treated as `tool.call_completed`, although it represents retry/failure feedback.                 | `_sdk_tool_result_to_raw()` now maps `part_kind="retry-prompt"` to raw `tool_call_failed` with `retryable=True`; regression test added.  |
| F2  | medium   | Builtin tool-call starts could be duplicated when both native `PartStartEvent[builtin-tool-call]` and deprecated `BuiltinToolCallEvent` were emitted. | Deprecated `builtin_tool_call` events are ignored; native part start is the single source for builtin tool start; regression test added. |

## Final Review

Verdict: PASS.

The reviewer confirmed:

- `retry-prompt` tool results now map to `tool_call_failed` with `retryable=True`.
- Deprecated `builtin_tool_call` events are ignored, while `part_start` with `part_kind="builtin-tool-call"` remains the single source for builtin tool starts.
- New tests cover both fixes.
- Capabilities remain narrow and truthful: streaming is enabled; tool/MCP/HITL/deferred/cancel/multimodal/evidence remain false or empty.
- Lazy import and ADR-0002 boundaries are preserved.
