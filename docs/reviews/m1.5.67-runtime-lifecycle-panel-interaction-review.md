## Result

## Initial Findings

- P1: `interaction.dispose()` incorrectly disposed the caller-provided `viewModel`, which would let a keyboard interaction layer tear down a shared panel view-model/adapter/controller in future wiring.
- P2: view-model operations double-published interaction notifications after upstream view-model methods had already published through the subscription.
- P2: `dispatch("activate_focused_command")` threw for disabled focused commands while shortcut dispatchers no-op, leaving disabled keyboard activation behavior inconsistent and under-tested.

## Fixes

- `interaction.dispose()` now unsubscribes from the view-model and releases only interaction-owned page lifecycle bindings; it does not dispose the injected view-model.
- `invokeCommand()`, `selectFocusedTimelineItem()`, `clearSelection()`, and shortcut command invocation now reconcile state and return snapshots without an extra `publish()`.
- `dispatch("activate_focused_command")` now no-ops for disabled/busy focused commands through the shortcut path, while direct `invokeCommand("wait")` remains fail-closed.
- Tests now assert active page lifecycle cleanup, non-ownership of the injected view-model, exact listener error count for duplicate-publication prevention, disabled focused command no-op, and direct disabled invocation failure.

## Review Notes

- The interaction layer remains pure TypeScript with no React/Electron import, new IPC channel, production dependency, spec/ADR/schema change, or generated-code contract drift.
- It consumes only the W1.5.66 view-model public interface and does not expose raw `viewModel`, `adapter`, `controller`, `session`, or `presenter` properties.
- Keyboard commands cover primary/adjacent command focus, focused command activation, refresh/stop/start-or-retry shortcuts, adjacent timeline focus, focused timeline selection, and clear selection.
- Runtime unsafe interaction commands, command ids, and timeline item ids fail closed.
- Listener failures and `onError` failures are isolated; unsubscribe, active dispose, page lifecycle cleanup, and disposed fail-closed behavior are covered.
