# W1.5.183 Skill Management Desktop Bridge Review

Reviewer: Euclid (`019eff28-2fd6-7421-beb2-46437d6f9af9`)
Date: 2026-06-25

## Scope

Review W1.5.183 runtime-backed Skill management bridge across runtime harness/API, desktop workbench session/interaction, React shell, tests, and runbook evidence. The review was read-only.

## Initial Findings

- P1: `PATCH /projects/{project_id}/skills` accepted arbitrary `params`, persisted them to git-tracked `skills.config.json`, and `GET /skills` returned full params values to the renderer-facing API surface.
- P1: Skill management reads used the forgiving list reader, so invalid JSON or non-list `skills.config.json` could be reported as an empty list and later overwritten by PATCH.
- P2: `CLAUDE.md` was dirty but unrelated to W1.5.183 and must not be included in the slice commit.

## Fixes Reviewed

- Split persisted Skill manifest entries from API management entries: internal `skills.config.json` entries may retain `params`, but GET/PATCH management responses expose only `skill_id`, `version`, and `enabled`.
- Removed `params` from `ProjectSkillPatchRequest`; existing manifest params are preserved on updates and new entries default to `{}`.
- Sanitized Pydantic validation error details to avoid echoing rejected request input values.
- Added strict Skill manifest reading for the management path: missing manifest is treated as empty, while invalid JSON, non-list JSON, and invalid entries return `RH_MANIFEST_REVISION_MISMATCH`.
- Updated Desktop parsing so Skill responses do not read or retain `params`; runtime session snapshots keep no raw params values.
- Added regression tests for params rejection, missing manifest, corrupted manifest without overwrite, and non-list manifest rejection.

## Delta Review

Euclid delta-reviewed the fix and reported PASS with no new blocker/P1/P2 findings:

- Skill API no longer exposes or accepts `params`; public entries contain only `skill_id`, `version`, and `enabled`.
- Rejected `params` values are covered without echoing raw input values.
- Desktop response parsing no longer reads `params`.
- Strict reader closes missing/corrupt/non-list behavior and PATCH reads before writing, so corrupted manifests are not overwritten.
- `CLAUDE.md` remains unrelated and unstaged.

The delta review noted a residual invalid-entry test gap, which was closed with an additional endpoint regression asserting invalid Skill entries fail with `RH_MANIFEST_REVISION_MISMATCH` without echoing raw values.
