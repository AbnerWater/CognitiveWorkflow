# M1.5.132 Runtime Stream Unknown Visual Smoke Review

## Scope

W1.5.132 is a dev-only visual smoke hardening slice for the Runtime Stream unknown-event UI path. It adds a query-parameter mode to the existing visual smoke fixture so the same Electron smoke runner can verify both the default known `model.text_delta` event and an unknown `adapter.experimental_event` event.

Files reviewed:

- `apps/desktop/src/renderer/runtime-workbench-shell-visual-smoke.tsx`
- `apps/desktop/scripts/runtime-workbench-visual-smoke.cjs`

Out of scope:

- Production runtime/session/store/React shell behavior.
- Accepted spec / ADR / schema / generated schema changes.
- Dependency, runtime API, IPC / `window.cw`, renderer storage, React Flow, Tailwind, packaging/updater changes.
- Raw payload, `rawData`, metadata object/key/value display.

## Independent Review

Reviewer: Sartre (`019ef955-87fb-7351-be07-725d96e81e90`)

Initial result: FAIL.

Finding:

- P2: the first version asserted unknown timeline and selected-event badges, but expanded event detail and expanded selection metadata only checked `knownType=false`; it did not assert the visible `Unknown event type` text in those surfaces.

Resolution:

- `runtime-workbench-visual-smoke.cjs` now collects `streamEventDetailText` and `streamSelectionMetadataText`.
- `expectedStreamEvent()` includes the mode-specific `typeStatusLabel`.
- The smoke runner asserts `Known event type` / `Unknown event type` in both expanded event detail and expanded selection metadata.
- Default known visual smoke and unknown visual smoke were rerun successfully.

Final result: PASS, no remaining findings.

The reviewer confirmed the diff remains limited to dev/test visual-smoke files, the default mode keeps known `model.text_delta`, the unknown mode uses `adapter.experimental_event`, and no forbidden production/spec/generated/dependency/API/IPC/storage/React Flow/Tailwind/packaging surface drift was found.
