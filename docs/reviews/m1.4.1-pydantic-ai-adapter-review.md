## Initial Findings

### High: PydanticAI capabilities widened beyond spec

The first implementation advertised `kinds={chat, model_only}`. `specs/protocols/agent_adapter.md §2.1` lists PydanticAI as `chat` only, so the extra `model_only` kind widened routing eligibility beyond the accepted capability snapshot.

Fix: `PydanticAIAdapter.capabilities()` now returns `kinds={AdapterKind.CHAT}`. The focused capability test and runbook were updated to lock the spec-aligned value.

### Medium: unterminated session streams left handles non-terminal

`run()` and `resume()` could fall off the raw session iterator without producing `attempt.completed` or `attempt.failed`. That left the handle in `running` state and made `finalize()` raise instead of returning a structured adapter outcome.

Fix: both paths now emit terminal `attempt.failed` with `AA_RUN_INTERNAL` when the session iterator ends without a terminal raw event. Regression tests cover empty run and resume session streams.
