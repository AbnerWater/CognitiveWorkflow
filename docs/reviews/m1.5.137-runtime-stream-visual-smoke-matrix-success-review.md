## Independent Review

Reviewer: Pasteur (`019ef9dd-b0a1-7d91-a16f-cde34834dffc`)

Initial result: P3 finding.

Finding:

- P3: the first success-path fake CLI derived mode from `CW_VISUAL_SMOKE_URL` but did not fail if the child URL still contained the original query/hash secret. The test proved secrets were absent from parent stdout/stderr/manifest, but not that the matrix runner stripped them before invoking the child.

Resolution:

- The fake CLI now writes `urlFailures` if `targetUrl.hash` is non-empty, if known mode still has any search params, or if unknown mode has anything except `?streamEvent=unknown`. This turns unstripped child URLs into matrix failures.

Final result: PASS, no blocker/P1/P2/P3 findings.

Pasteur confirmed the success-path test uses `CW_VISUAL_SMOKE_ELECTRON_CLI` with a fake CLI, executes the real matrix script as a child process, and verifies the 5-case manifest without real Electron/Vite startup or external port dependency. The reviewer also confirmed the diff is test/documentation only and does not introduce production/spec/generated/dependency/API/IPC/window.cw/storage/React Flow/Tailwind/packaging/updater drift.

The new success-path regression verifies:

- Matrix exit code is 0 for valid fake evidence.
- Manifest has 5 cases and `failures=[]`.
- Known desktop and unknown desktop modes are both exercised.
- Deep-scroll evidence records observed `y=1440` and `maxY=2000`.
- Manifest target location strips the input URL query/hash secret.
