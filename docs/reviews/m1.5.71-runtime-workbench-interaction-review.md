## Confirmed Properties

- The slice stays limited to renderer-side pure TypeScript and does not change dependencies, specs, ADRs, schema models, IPC channels, or generated code.
- `createRuntimeWorkbenchInteraction()` exposes the expected external-store shape: `getSnapshot()`, `getServerSnapshot()`, `snapshot()`, `subscribe()`, `dispatch()`, `setActivePanel()`, `listenerCount()`, `dispose()`, and `isDisposed()`.
- Lifecycle panel commands route through the narrow public `workbench.dispatchLifecyclePanelCommand()` API instead of reaching into raw controllers or sessions from the interaction layer.
- The interaction snapshot exposes data snapshots and command ids only; it does not expose raw workbench, controller, session, factory, adapter, viewModel, or stream internals.
- Open failure retains old state; invalid top-level and nested commands fail closed; inactive lifecycle command dispatch fails closed.
- Listener and `onError` failures are isolated.
- No-op active panel dispatch does not publish or replace snapshot identity.
- Interaction dispose releases its own subscription and listeners while leaving the injected workbench owned by its caller.
