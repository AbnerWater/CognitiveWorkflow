## Initial Finding

### High: capabilities advertised unimplemented routing features

The first W1.4.2 implementation added a default SDK-backed session but still advertised `tool_call`, `mcp`, `human_in_the_loop`, `deferred_tool_results`, `cancel`, `evidence_lookup_tool`, and multimodal support. The default SDK session only built `Agent(..., output_type=StructuredDict(...))` and called `agent.run(...)`; it did not build Toolsets/MCP/HITL/deferred resume/cancel semantics.

Because ModelRouter filters before `prepare()` using `capabilities()`, those flags could route unsupported nodes to PydanticAIAdapter.

Fix: `PydanticAIAdapter.capabilities()` now declares only the W1.4.2 implemented route surface: `chat`, structured output, provider kinds, and model settings passthrough. Streaming, tool calls, MCP, HITL, deferred tool results, multimodal input, cancel, and evidence lookup stay false/empty until later M1.4 slices implement those semantics.
