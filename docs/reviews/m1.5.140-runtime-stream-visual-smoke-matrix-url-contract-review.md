# M1.5.140 Runtime Stream Visual Smoke Matrix URL Contract Review

## Scope

W1.5.140 adds automated regression coverage for the dev-only Runtime Stream
visual smoke matrix URL startup contract. The tests cover matrix URL
precedence, legacy single-case URL fallback, and fail-fast behavior when no
target URL is configured.

Files reviewed:

- `apps/desktop/scripts/runtime-workbench-visual-smoke-matrix.test.cjs`
- `docs/04_runbook/m1.5-progress.md`

Out of scope:

- Production runtime/session/store/React shell behavior.
- Accepted spec / ADR / schema / generated schema changes.
- Dependency, runtime API, IPC / `window.cw`, renderer storage, React Flow,
  Tailwind, packaging/updater changes.
- Raw payload, `rawData`, metadata object/key/value display.

## Independent Review

Reviewer: James (`019efa3a-2e4a-7041-b97e-0a75515160fe`)

Initial result: FAIL with one P2 and one P3 finding.

- P2: matrix-only and legacy-only tests did not cover the precedence contract
  when both `CW_VISUAL_SMOKE_MATRIX_URL` and `CW_VISUAL_SMOKE_URL` are set.
  A regression that reversed precedence could still pass.
- P3: runbook wording said missing URL stderr only contains the generic
  required-env message, while the runner emits a Node stack trace at module
  load. The test only requires the generic message and no secret leakage.

Fixes applied:

- Added `visual smoke matrix prefers matrix URL over legacy URL`, with distinct
  matrix and legacy target URLs. The fake Electron CLI validates the actual
  per-case URL origin/path, and the manifest is asserted to use the matrix
  target while omitting query/hash secrets from both configured URLs.
- Updated the runbook wording to state that missing URL stderr includes the
  generic required-env message and no secrets.

Final result: PASS, no blocker/P1/P2/P3 findings.

James confirmed:

- The P2 precedence gap is closed.
- The P3 runbook wording is aligned with runner behavior and assertions.
- The diff remains limited to the target test and runbook files before this
  review artifact was added.
