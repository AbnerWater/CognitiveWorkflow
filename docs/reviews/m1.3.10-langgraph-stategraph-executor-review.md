# M1.3.10 LangGraph StateGraph Executor Review

## Scope

W1.3.10 adds the first real LangGraph StateGraph executor foundation:

- `compile_langgraph_state_graph()` compiles `EngineWorkflowIR` into a LangGraph `StateGraph`.
- Each `EngineNode` maps to one LangGraph node; evaluation / repair / human routes remain Engine IR route keys, not nested subgraphs.
- Node work is delegated through `LangGraphNodeExecutor`, keeping Adapter calls outside this slice.
- LangGraph is lazy-imported inside `cw_runtime.engine` so default runtime imports do not require the optional `graph` extra.

## Independent Review

Initial verdict: `FAIL`.

Findings:

- Explicit `LangGraphNodeResult.next_node_ids` could combine targets from multiple compiled routes and bypass route-key semantics.
- Runbook marked W1.3.10 done while its review section was still pending.
- `CLAUDE.md` remained dirty and must stay out of the W1.3.10 commit.

Fixes:

- Explicit `next_node_ids` now must equal exactly one compiled route target set; when `route_key` is supplied, explicit targets must match that route.
- Added regression coverage rejecting a delegate that combines `pass` and `fail` targets.
- Runbook was moved to `in review` before re-review and should be marked done only after PASS.

Follow-up verdict: `PASS`.

No remaining implementation or spec-alignment findings. `CLAUDE.md` remains dirty but unstaged and must be excluded from commit scope.
