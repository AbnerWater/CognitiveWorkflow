## Scope

Reviewed the W1.4.8 narrow slice against `specs/schemas/node_contract.md`, `specs/protocols/agent_adapter.md`, `specs/schemas/stream_event.md`, and `docs/04_runbook/m1.4-progress.md`.

The review focused on:

- Whether `ExecutionPack.retry_policy.timeout_seconds` is projected into the Pydantic AI adapter run request.
- Whether `run()` and `resume()` enforce an adapter-owned attempt deadline outside the Pydantic AI session iterator.
- Whether timeout uses existing spec vocabulary: `AA_RUN_CANCELLED`, `CancelReason.IDLE_TIMEOUT`, spec-allowed `attempt.failed`, and `AttemptOutcome.state=CANCELLED`.
- Whether SDK/tool-level `TimeoutError` remains distinct from CW attempt timeout.
- Whether `capabilities().cancel` remains truthful and stays `False`.

## First Review

Verdict: FAIL.

Finding:

| ID  | Severity | Description                                                                                                                                                                                                       | Resolution                                                                                                                                                                                           |
| --- | -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| F1  | high     | `_iterate_session()` caught any `TimeoutError` from the session iterator and converted it to `AA_RUN_CANCELLED`, so SDK/tool/network timeouts could be misclassified as CW `retry_policy.timeout_seconds` expiry. | Adapter now uses an internal `_AttemptTimeoutExpired` sentinel and `asyncio.ensure_future` plus `asyncio.wait`; SDK/tool `TimeoutError` no longer triggers cancellation and has regression coverage. |

## Final Review

Verdict: PASS.

The reviewer confirmed:

- Adapter-owned timeout expiry now maps to `AA_RUN_CANCELLED` with `CancelReason.IDLE_TIMEOUT`.
- The timeout path emits spec-allowed `attempt.failed` and finalizes `AttemptOutcome.state=CANCELLED`.
- Plain SDK/tool `TimeoutError` no longer triggers cancellation.
- `capabilities().cancel=False` is preserved.
- No accepted specs, ADRs, or production dependencies were changed.

Residual gap: tests use injected fake sessions rather than a real Pydantic AI SDK streaming run. This matches the W1.4.8 slice boundary and the optional dependency stance.
