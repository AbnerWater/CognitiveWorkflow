## Initial Review Result

FAIL.

Blocking issue:

- `_migrate_project_secret_table()` accepted composite primary keys such as `PRIMARY KEY(secret_id, scope)`.
- That schema is incompatible with `runtime_harness.md §6` because `secret_id` must be the table primary key, and the write path uses `ON CONFLICT(secret_id)`.

## Repair

- Tightened migration validation so `secret_id` must have primary-key position `1`.
- Reject any non-`secret_id` column with a non-zero primary-key position.
- Added `test_project_secret_store_migration_rejects_composite_primary_key()` for `PRIMARY KEY(secret_id, scope)`.
