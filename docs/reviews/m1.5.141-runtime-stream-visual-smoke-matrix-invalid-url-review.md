# M1.5.141 Runtime Stream Visual Smoke Matrix Invalid URL Review

## Scope

W1.5.141 hardens the dev-only Runtime Stream visual smoke matrix invalid URL
path. The runner now validates the configured matrix base URL before creating
the output directory, running any Electron smoke case, or writing
`matrix.json`, and replaces Node's raw invalid URL error with a fixed safe
message.

Files reviewed:

- `apps/desktop/scripts/runtime-workbench-visual-smoke-matrix.cjs`
- `apps/desktop/scripts/runtime-workbench-visual-smoke-matrix.test.cjs`
- `docs/04_runbook/m1.5-progress.md`

Out of scope:

- Production runtime/session/store/React shell behavior.
- Accepted spec / ADR / schema / generated schema changes.
- Dependency, runtime API, IPC / `window.cw`, renderer storage, React Flow,
  Tailwind, packaging/updater changes.
- Raw payload, `rawData`, metadata object/key/value display.

Bernoulli confirmed:

- Invalid URL validation happens before output directory creation, case loop, or
  manifest write.
- Invalid URL errors are replaced with a fixed message and do not retain the
  original Node URL error object.
- Case URL generation still strips query/hash and preserves safe origin/path
  manifest behavior.
- The test helper deletes inherited parent URL env vars before setting
  matrix/legacy modes.
- The new invalid URL regression checks fail-fast behavior, no fake CLI run, no
  manifest, and no secret fragments in stderr.

Residual test gap:

- Invalid legacy-only URL and invalid matrix URL with valid legacy fallback are
  covered by the shared `parseBaseUrl()` path but do not have separate explicit
  regression cases in this slice.
