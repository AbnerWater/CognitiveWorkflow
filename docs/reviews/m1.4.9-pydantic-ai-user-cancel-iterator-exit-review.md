## Scope

Reviewed the W1.4.9 narrow slice against `specs/protocols/agent_adapter.md`, `specs/schemas/stream_event.md`, `specs/schemas/node_contract.md`, and `docs/04_runbook/m1.4-progress.md`.

The review focused on:

- Whether `cancel()` wakes a `run()` iterator blocked waiting for the next session raw event.
- Whether `cancel()` wakes a `resume()` iterator blocked after `human.gate_resolved`.
- Whether the W1.4.8 `retry_policy.timeout_seconds` path remains adapter-owned and distinct from SDK/tool `TimeoutError`.
- Whether the slice uses existing spec vocabulary: `AA_RUN_CANCELLED`, `CancelReason`, and `attempt.failed`.
- Whether `capabilities().cancel` remains truthful while real SDK-backed upstream cancellation is unproven.

## Review

Verdict: PASS.

The reviewer confirmed:

- The diff stays limited to the adapter, focused tests, and M1.4 runbook.
- No accepted specs, ADRs, or production dependencies were changed.
- `capabilities().cancel=False` is preserved.
- User cancel uses `AA_RUN_CANCELLED` and `CancelReason`, without inventing a StreamEvent type outside `stream_event.md`.
- The W1.4.8 timeout sentinel remains in place, and SDK/tool `TimeoutError` is not misclassified as CW cancellation.

Residual gaps: real SDK-backed upstream cancellation is not proven, and the default SDK session cancellation remains a no-op. This matches the W1.4.9 boundary and retained `capabilities().cancel=False`.
