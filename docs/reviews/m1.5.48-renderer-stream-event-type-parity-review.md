## Review Notes

- `RuntimeStreamGeneratedEventType` is derived from generated `@cw/schemas` event interfaces through their `type` fields.
- The two-way `Exclude` parity guard covers generated event type additions, removals, and renames when the renderer constants are not synchronized.
- The `@cw/schemas` imports are type-only and introduce no runtime import.
- Tests now cover all-known event type uniqueness, exact run/planning defaults, default event subset membership, retained non-SSE-default system event types, and unsafe override fail-closed behavior.
- The slice remains pure TypeScript renderer work: no Electron, React, Vite, Tailwind, storage API, IPC channel, package/lockfile, accepted spec, ADR, schema, or codegen behavior change.

## Residual Risks

- The guard aligns against generated TypeScript schemas, not the Python source directly, so it relies on `codegen` staying part of the gate.
- If a future spec change adds an entirely new StreamEvent category/interface, the `RuntimeStreamGeneratedEventType` interface list must be extended in addition to the event constants.
- This slice still does not validate real Electron/browser EventSource behavior, live sidecar replay, React rendering, actual full reload UI rebuild, cross-window sharing, runtime StreamEvent schema validation, or automatic generation of renderer session constants.
