# M1.5.134 Runtime Stream Visual Smoke Matrix Review

## Scope

W1.5.134 turns the W1.5.133 manual responsive visual-smoke loop into a dev-only matrix runner and manifest. It keeps the existing single-case Electron visual smoke runner as the source of truth for UI assertions, then validates and summarizes known/unknown responsive cases in one JSON manifest.

Files reviewed:

- `apps/desktop/scripts/runtime-workbench-visual-smoke.cjs`
- `apps/desktop/scripts/runtime-workbench-visual-smoke-matrix.cjs`
- `apps/desktop/package.json`
- `docs/04_runbook/m1.5-progress.md`

Out of scope:

- Production runtime/session/store/React shell behavior.
- Accepted spec / ADR / schema / generated schema changes.
- Dependency, runtime API, IPC / `window.cw`, renderer storage, React Flow, Tailwind, packaging/updater changes.
- Raw payload, `rawData`, metadata object/key/value display.

## Independent Review

Reviewer: Pascal (`019ef98d-84a0-7253-aeab-2d0e480b01ea`)

Initial result: CONDITIONAL_PASS.

Findings:

- P2: scrolled matrix cases only proved requested scroll input, not observed page scroll. If `window.scrollTo()` no-ops or clamps unexpectedly, `unknown-mobile-scroll-900` and `unknown-mobile-scroll-1440` could still pass.
- P2: W1.5.134 was marked done in the top progress table without a matching runbook body and review artifact.

Resolution:

- `runtime-workbench-visual-smoke.cjs` now records `metrics.scroll` with `x`, `y`, and `maxY`.
- `runtime-workbench-visual-smoke-matrix.cjs` asserts observed scroll equals `min(requestedScrollY, observedMaxScrollY)` and includes `observedScroll` in each manifest case summary.
- W1.5.134 runbook scope, plan, verification results, and this review artifact were added.
- Matrix rerun after the fix proved actual scroll positions: `unknown-mobile-scroll-900` observed `900/5750`, `unknown-mobile-scroll-1440` observed `1440/5750`, and manifest `failures=[]`.

Final result: PASS, no remaining findings.

The reviewer confirmed both prior P2s are closed: scroll evidence is now real observed state, and W1.5.134 has sufficient runbook/review closure evidence. The reviewer also confirmed the runner is dev-only, uses `spawn(process.execPath, [electronCliPath, smokeScriptPath])` without shell execution, and did not find production/session/store/schema/API/IPC/dependency drift. URL handling strips query/hash for case execution and writes safe target fields only.
