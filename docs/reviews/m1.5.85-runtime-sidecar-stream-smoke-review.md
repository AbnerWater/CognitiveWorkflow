## Independent Review

Reviewer: James (`019ef07d-8c31-7d33-bfbc-837fecf0bbbf`)

Findings:

- P2: initial cleanup awaited desktop ownership cleanup but did not wait for the Python sidecar process to exit before removing the temporary project directory. On Windows/CI this could race open runtime files or leave the sidecar alive briefly across test runs.

Resolution:

- Fixed in `runtime-sidecar-stream-smoke.test.ts` by capturing `startup.session.sidecar.closed` before stop, awaiting `startup.stop("SIGTERM")`, awaiting `startup.closed`, and then waiting up to 5 seconds for the sidecar process to exit before deleting the temporary directory.
- The timeout helper clears its timer on success so the Node test process is not delayed by a stale timer.
