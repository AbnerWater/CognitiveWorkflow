## Review Notes

- Runtime Chat instruction dispatch now uses the accepted `RuntimeInstructionRequest` / `RuntimeInstructionAccepted` contract instead of overloading FR-007 `run-once`.
- Runtime persists only command metadata; raw instruction text is present in the authenticated request body and is not copied into the JSONL projection.
- Artifact open/download runs through preload/main IPC. Main fetches artifact content from the runtime content endpoint, writes bytes to a privileged temp file, and returns only sanitized `ArtifactActionResult` metadata.
- Main does not return raw artifact bytes or absolute destination paths to renderer; renderer snapshots keep status, byte count, hash, content type, sensitivity, and destination kind only.
- The implementation intentionally does not invent unregistered concrete `error_code` values for local Desktop failures; status and destination kind carry the observable result until a spec-owned code table exists.
- FR-008 and FR-017 are implemented enough for follow-up evidence capture, but they are not marked accepted in this slice. `EXIT-P1-1` remains `not_ready`.
