## Findings

### P3: W1.5.90 visual-smoke plan was not runnable as written

Initial W1.5.90 runbook verification listed a bare `pnpm --filter @cw/desktop run visual-smoke`, but `runtime-workbench-visual-smoke.cjs` requires `CW_VISUAL_SMOKE_URL` and `CW_VISUAL_SMOKE_OUTPUT`.

Resolution: W1.5.90 verification now documents the required environment variables and separate desktop, mobile top, and mobile drawer smoke invocations.

## Accepted Checks

- Drift: no dependency, spec, ADR, schema, generated schema, or IPC contract diff was introduced.
- DTO boundary: Task Drawer collapse copy and defaults are presenter-derived from existing active panel, visible count, and unread count state.
- Adapter isolation: adapter clone/freezes the expanded Task Drawer DTO fields before exposing the snapshot.
- React boundary: collapse/expand state is local UI state only and does not add runtime commands or access host/session/controller/store internals.
- Responsive behavior: the Task Drawer header falls back to a single column at narrow widths to avoid button/title overlap.
- Visual smoke: the runner clicks the Task Drawer toggle, validates collapsed and expanded DOM metrics, keeps desktop viewport validation, and records both `metrics` and `collapsedMetrics`.
