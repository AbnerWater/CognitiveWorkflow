## Repaired Finding

- Initial independent review found a high-priority issue: malformed renderer-supplied `init` values could reach `cw:runtime:fetch` because the bridge used the typed fetch request builder directly.
- The repair validates the composed `{ path, init }` payload with the shared unknown parser before invoking IPC.
- The repair adds malformed `init` and non-string `method` tests, and asserts invalid fetch payloads do not call the injected invoke function.
