- Date: 2026-06-18

- Reviewer: independent subagent Hilbert

## Initial Findings

- FAIL: `edit` moved the run directly to the `routing_map.edit` target while the human checkpoint node remained `retrying`, bypassing the `retrying -> running` scheduling step required by `workflow_run.md`.
- FAIL: `reject` transitioned the human checkpoint node to `failed` and the run to `failed` without first emitting an `attempt.failed` lifecycle event.

## Fixes

- `edit` now leaves `current_node_ids=[human_node_id]`, stores `user_edit` resumption metadata, and requires the next `advance_workflow_run()` call to consume the resumption before routing to the declared `edit` target.
- `reject` now emits `attempt.failed` with `error_kind`, `will_retry`, and `next_action` before `node.state_changed` and `run.failed`.
