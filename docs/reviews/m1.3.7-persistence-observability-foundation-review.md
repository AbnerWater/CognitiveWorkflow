## Implementation Summary

- Runtime creates a derived SQLite index at `.agent-workflow/cache/runtime_index.sqlite`.
- Runtime creates the local OTel exporter database at `.agent-workflow/traces/trace.sqlite` with the spec `spans / resources / metrics` tables.
- StreamEvent append updates the runtime index and projects public events according to `observability.md §5.1` without copying payload text into OTel attributes.
- `model.thinking_delta / model.text_delta` remain in the runtime index but do not create independent spans.
- `metric.snapshot / usage.delta` project to the `metrics` table, not independent spans.
- Run JSONL append updates a derived `runtime_jsonl_records` index.
- Project initialization records the initial git commit in `snapshots/snapshots.jsonl`.
- `run.started`, `attempt.completed`, and terminal run events trigger runtime git snapshots; terminal runs also create `run-<run_id>-<state>` tags.

## Initial Findings

Independent review initially returned `FAIL` with required fixes:

- `trace_id` used `run_id` rather than `StreamEvent.correlation_id`.
- `model.thinking_delta / model.text_delta` and `metric.snapshot / usage.delta` were incorrectly projected as independent spans.
- `resources.resource_hash` used blake2b rather than spec-required blake3.
- W1.3.7 docs/review status were premature.

## Fixes Verified

- OTel `trace_id` now uses 32-hex `correlation_id` exactly when present; non-hex correlation IDs and null fallback are blake3-derived.
- Model delta events are indexed but skipped by the independent span exporter.
- Metric and usage events write the `metrics` table only.
- Resource hashes use blake3 over canonical JSON.
- `blake3` is lazy-imported at hash time so importing the base harness does not require the runtime extra.
- Regression coverage was added in `test_trace_exporter_uses_correlation_id_and_skips_delta_or_metric_spans`.

## Residual Boundaries

- SQLite indexes are derived cache/query views; JSON/JSONL remains authoritative.
- No SQLCipher secure trace store, remote OTLP exporter, GC, full LangGraph executor, or real Adapter trace is implemented in this slice.
- Latest snapshot metadata is appended after the commit it references; the next snapshot commit persists the previous record.
