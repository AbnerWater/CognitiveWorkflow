## Review Notes

- The session controller layer remains pure TypeScript with no React/Electron import, new IPC channel, production dependency, spec/ADR/schema change, or generated-code contract drift.
- `createRuntimeLifecyclePanelSessionController()` exposes a React external-store style shape through `getSnapshot()`, `getServerSnapshot()`, `snapshot()`, and `subscribe(() => void)` without importing React.
- Snapshot identity is cached by signature and remains stable when state has not changed.
- Active session replacement, create failure, active disposal, controller dispose, unsubscribe, and disposed fail-closed paths are idempotent.
- Old session subscriptions are released before active session replacement; create failure keeps the previous active session.
- Listener failures and `onError` failures are isolated.
- The controller object does not expose raw `factory`, `controller`, `adapter`, `viewModel`, or `interaction` properties; the interaction value in snapshots is snapshot data.
- Tests cover active session replacement, create failure retention, active session publish, old session subscription release, active disposal publish, controller dispose publish, listener/onError isolation, unsubscribe idempotency, and disposed fail-closed behavior.
