## Findings

- P3 fixed: `RuntimeStreamInteractionSession.dispatch()` previously called `interaction.dispatch(command)` and then `publish()`, while the session subscription already forwards `interaction.subscribe()` updates into `publish()`. The extra call could notify direct session/controller subscribers twice for one command and duplicate listener error reports. The fix removes the second publish and asserts one command dispatch emits one unified session snapshot.

## Confirmed Properties

- Stream interaction state exposes a typed `RuntimeStreamInteractionCommand` dispatch surface for search, match navigation, selection, expansion, read state, and full-reload acknowledgement.
- Workbench/session layers route the commands through `dispatch_runtime_stream`; the command is enabled only when a runtime stream session is active.
- React controls emit command DTOs through the existing shell session dispatch path and do not access raw stream internals.
- The shell projection remains display-only data: no raw `store`, `viewModel`, `interaction`, raw SSE data, payload, rawData, or artifact refs are rendered through shell/React.
- This slice intentionally does not add global keyboard shortcuts for stream controls.
