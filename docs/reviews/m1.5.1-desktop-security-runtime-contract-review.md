# W1.5.1 Desktop Security Runtime Contract Review

## Scope

- Review M1.5.1 desktop foundation diff.
- Confirm no accepted spec, ADR, public schema, or production dependency drift.
- Confirm the first desktop slice stays within security/runtime contract boundaries and does not overclaim runnable Electron UI completion.
- Confirm tests cover sidecar READY parsing, loopback runtime URL construction, token/header injection, CSP defaults, and BrowserWindow security defaults.

## Review Result

PASS.

The independent subagent confirmed:

- `apps/desktop/package.json` adds real build/typecheck/test/lint scripts without adding production dependencies.
- `apps/desktop/src/main/runtime.ts` parses only `READY <port>` into `http://127.0.0.1:<port>/cw/v1` and rejects malformed ports or URL-shaped READY lines.
- `apps/desktop/src/main/security.ts` locks BrowserWindow security defaults to `contextIsolation=true`, `sandbox=true`, `nodeIntegration=false`, `webSecurity=true`, and `allowRunningInsecureContent=false`.
- `apps/desktop/src/main/security.ts` builds a strict CSP without `unsafe-eval`, `unsafe-inline`, `localhost`, `0.0.0.0`, or wildcard HTTP sources.
- `apps/desktop/src/preload/contract.ts` defines the `window.cw.runtime` preload contract, injects runtime auth/client headers, blocks renderer override of reserved headers, and rejects whitespace/control-character tokens.
- `docs/04_runbook/m1.5-progress.md` does not mark M1.5 complete and explicitly excludes real Electron BrowserWindow, IPC, sidecar spawn, EventSource, React renderer, and Electron Forge/Vite/Tailwind wiring from W1.5.1.
