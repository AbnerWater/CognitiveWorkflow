## Initial Finding

- Minor: the first presenter interface exposed a `session` property. The TypeScript type was narrowed to a safe `Pick`, but the runtime value returned the original session object, which left a future UI layer a path to bypass the presenter projection.

## Fix

- Removed `RuntimeLifecyclePanelPresenter.session` from the public interface and returned presenter object.
- Added tests for `retry_startup` command sequencing, throwing `onError` isolation, and active-listener dispose cleanup.

## Review Notes

- The presenter remains a pure TypeScript mapping layer with no React/Electron import, new IPC channel, production dependency, spec/ADR/schema change, or generated-code contract change.
- Panel-facing snapshots are built only from the W1.5.62 shell session safe view projection.
- The command contract only calls the existing session `start()` / `refresh()` / `stop()` tracking operations; it does not claim real sidecar UI wiring.
- Listener snapshots are independently built, listener failures are isolated, and `onError` failures do not block other listeners.
- Presenter `dispose()` is idempotent, cleans presenter-to-session subscriptions while active listeners exist, and does not dispose the externally owned session.
