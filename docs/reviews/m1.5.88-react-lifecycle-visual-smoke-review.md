## Findings

### P2: Visual smoke runner recorded bad metrics without failing

Initial runner wrote JSON metrics and PNG output but only failed on an empty screenshot. Regressions such as horizontal overflow, Vite overlay text, wrong timeline or command counts, and captured warning/error console messages could still exit successfully.

Resolution: `runtime-workbench-visual-smoke.cjs` now validates the collected metrics and warning/error messages, writes `failures` into the JSON evidence, and exits nonzero when any visual smoke invariant fails.

### P3: Runbook hard-coded the local Vite fallback port

The initial runbook example used `http://127.0.0.1:5174/visual-smoke.html`, which only matched the local fallback port in this workspace. A clean environment may use Vite's default `5173`.

Resolution: the runbook now instructs operators to use the Local URL printed by Vite and append `/visual-smoke.html`.

## Accepted Checks

- Dev-only/default entry: `index.html` still loads the normal `/src/renderer/main.tsx`; the visual smoke route is only reachable by explicitly loading `visual-smoke.html`.
- Electron runner safety: the screenshot window uses `contextIsolation: true`, `nodeIntegration: false`, and `sandbox: true`.
- Snapshot stability: the fixture caches `currentSnapshot` and `getSnapshot()` returns the cached object for React external-store semantics.
- Lifecycle commands: the runner clicks existing lifecycle buttons; React dispatches through the existing `dispatch_lifecycle_panel` facade.
- Responsive evidence: desktop, mobile top, and mobile timeline screenshots reported `horizontalOverflow: 0`, no Vite overlay, no page warning/error messages, and expected lifecycle DOM counts.
- Drift: no dependency, spec, ADR, schema, or codegen contract diff was introduced.
