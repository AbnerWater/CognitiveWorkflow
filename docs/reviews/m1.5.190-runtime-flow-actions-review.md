## Findings

1. P1 - Chat Box runtime routing drops the user instruction before runtime.

   FR-008 requires fixed-bottom Chat Box instructions to target the current node or global Workflow. The current React command only carries enum/count metadata, and `submitChatCommand()` calls `POST /runs/{run_id}/nodes/{node_id}:run-once` without a body. This preserves the unsafe-evidence boundary, but the sidecar receives no submitted instruction and no runtime-visible intent/target/action command payload. The current tests assert body absence and redaction, but not real runtime command semantics.

2. P1 - Artifact Open/Download reports a successful action surface without opening or downloading.

   FR-017 requires safe, project-scoped artifact open/download behavior. The current UI dispatches `open_artifact` / `download_artifact`, and `runArtifactAction()` only performs `GET /artifacts/{artifact_id}/content` then records status/header metadata. There is no save/open handoff, object URL, native shell action, or observable downloaded file. The tests prove the snapshot omits response body, but not that an open/download action occurs.
