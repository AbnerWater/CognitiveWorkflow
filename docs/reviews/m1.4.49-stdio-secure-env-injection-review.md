## Findings And Repair

- Initial review found P2: the fake stdio server env log asserted only selected uppercase/underscore header-like env variants, so a regression that passed exact `Authorization` or `X-CW-MCP-Token` into stdio env could slip through.
- Repair added exact `Authorization` and `X-CW-MCP-Token` env logging plus assertions after both discovery lock snapshot and concrete `tools/call` invocation.
- Re-review confirmed the P2 was closed and that `build_project_mcp_stdio_discovery_client_factory()` passes only `ProjectMCPSecretMaterial.env` into `ProjectMCPStdioDiscoveryClient`.
- Re-review residual noted a possible host-env flake if the shell already defined those header-like env names. Repair added `monkeypatch.delenv(...)` for every asserted env key before starting the fake stdio subprocess.
- Final re-review confirmed the host-env flake risk was closed and found no new actionable issue.

## Evidence

- The diff is limited to runtime harness code, project harness tests, the M1.4 runbook, and this review note.
- No accepted spec, ADR, public schema, or dependency file was modified.
- `ProjectMCPStdioDiscoveryClient(secret_env=...)` overlays validated secret env onto the stdio child process environment only when explicit secure-store backed material is supplied.
- `build_project_mcp_stdio_discovery_client_factory()` reuses `load_project_mcp_secret_material()` and passes only `material.env` to the stdio client.
- Coverage includes encrypted secure-store envelope decryption, stdio client factory construction, discovery lock snapshot, concrete `tools/call` invocation, header material non-injection, invalid env-name fail-closed behavior, and host-env-stable assertions.

## Boundary Check

- This slice implements explicit secure-store backed env injection for MCP stdio discovery/invocation clients.
- It does not implement OAuth refresh/token exchange, new HTTP behavior beyond W1.4.48, AdapterRegistry/API wiring, public or third-party provider validation, real provider upstream `cancel()`, Claude Agent SDK real-provider HITL, or end-to-end runtime API MCP invocation.

- NUL env-name and NUL env-value rejection are safe by inspection through the shared validator; the added direct regression test covers the `BAD=NAME` invalid-name path.
