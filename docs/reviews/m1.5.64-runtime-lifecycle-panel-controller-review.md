## Initial Finding

- P2: `getSnapshot()` originally computed a fresh object through `presenter.snapshot()` on every call. A future React `useSyncExternalStore` hook would observe a changed snapshot identity even when state had not changed.

## Fix

- Added `currentSnapshot` caching to the controller read path.
- Refreshed the cached snapshot on presenter notifications, command completion, and dispose.
- Kept listener publication isolated by passing a fresh `buildSnapshot()` result to each listener.
- Added contract tests for stable `getSnapshot()` / `snapshot()` identity and identity changes after start, refresh, live status, pagehide, and dispose.
