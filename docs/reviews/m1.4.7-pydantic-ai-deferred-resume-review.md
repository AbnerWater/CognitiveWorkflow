## Scope

Reviewed the W1.4.7 narrow slice against `specs/protocols/agent_adapter.md`, `specs/schemas/node_contract.md`, ADR-0002, and `docs/04_runbook/m1.4-progress.md`.

The review focused on:

- Whether the SDK-backed Pydantic AI session can translate `DeferredToolRequests` into CW waiting events.
- Whether deferred tool calls and approvals resume with SDK `DeferredToolResults`.
- Whether `capabilities()` advertises `human_in_the_loop` and `deferred_tool_results` only when a toolset factory is configured.
- Whether malformed deferred resume payloads fail closed with `AA_RESUME_INVALID_KIND`.
- Whether the slice avoids real registry, cancel, cost, model HTTP retry, and Claude SDK work.

## First Review

Verdict: FAIL.

Finding:

| ID  | Severity | Description                                                                                                                                   | Resolution                                                                                                                                            |
| --- | -------- | --------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| F1  | high     | Deferred resume payloads were validated by shape only; unknown ids, partial results, and call/approval kind mismatches could leak to the SDK. | Adapter now validates resume ids, result kind, and full pending coverage against stored SDK `DeferredToolRequests`; regression tests cover all cases. |

## Second Review

Verdict: FAIL.

Finding:

| ID  | Severity | Description                                                                                                                                       | Resolution                                                                                                                                |
| --- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| F2  | high     | `human_decision` resume for mixed pending calls and approvals sent approval-only `DeferredToolResults`, leaving pending call results for the SDK. | `human_decision` now fail-closes with `AA_RESUME_INVALID_KIND` when pending calls remain; mixed requests must use `deferred_tool` resume. |

## Final Review

Verdict: PASS.

The reviewer confirmed:

- SDK `DeferredToolRequests` are translated into CW tool/human waiting events.
- SDK resume receives `message_history` plus complete `DeferredToolResults`.
- Unknown deferred ids, partial results, result-kind mismatches, and mixed calls plus approvals via `human_decision` fail closed.
- No production dependency changes were introduced.
- ADR-0002 holds: Pydantic AI remains a lazy import inside the adapter boundary.
