# W1.5.2 Desktop Sidecar Lifecycle Review

## Scope

- Review W1.5.2 desktop sidecar lifecycle diff.
- Confirm no accepted spec, ADR, public schema, or production dependency drift.
- Confirm sidecar spawn/token/READY behavior aligns with M1.5, ADR-0006, and `http_sse.md §7`.
- Confirm runbook boundaries do not overclaim Electron IPC, renderer, packaged runtime path discovery, or runtime.lock reuse.

## Review Result

PASS.

The independent subagent confirmed:

- `generateRuntimeAuthToken()` produces a 16-byte base64 token by default.
- `buildRuntimeSidecarArgs()` rejects caller-owned `--http-port` / `--http-port=<n>` and appends desktop-owned `--http-port=0`.
- `buildRuntimeSidecarEnvironment()` injects `CW_RUNTIME_AUTH_TOKEN` without mutating the caller's base environment and rejects unsafe tokens.
- `startRuntimeSidecar()` uses `child_process.spawn` with `windowsHide=true` and stdout/stderr pipes, waits for line-delimited `READY <port>`, resolves `RuntimeConnectionInfo`, and fails closed on timeout, error, or pre-READY exit.
- Timeout cleanup kills the child with `SIGTERM`; pre-READY exit/error paths reject without leaking token material.
- `docs/04_runbook/m1.5-progress.md` leaves runtime.lock reuse, Electron IPC, renderer wiring, packaged runtime path discovery, EventSource/fetch, and cross-platform packaging to later slices.
