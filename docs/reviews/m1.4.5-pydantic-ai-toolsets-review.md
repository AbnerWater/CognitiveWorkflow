# W1.4.5 Pydantic AI Toolsets Review

| Item        | Value                                                       |
| ----------- | ----------------------------------------------------------- |
| Milestone   | M1.4 AgentAdapter                                           |
| Work item   | W1.4.5 PydanticAIAdapter SDK toolsets bridge                |
| Reviewed at | 2026-06-18                                                  |
| Reviewer    | independent subagent `019ed702-f896-7611-b29f-034b3bcaa3db` |
| Verdict     | PASS                                                        |

## Scope

Reviewed the W1.4.5 narrow slice against `specs/protocols/agent_adapter.md`, `specs/schemas/node_contract.md`, ADR-0002, and `docs/04_runbook/m1.4-progress.md`.

The review focused on:

- Whether `ExecutionPack.effective_toolsets` and `NodeContract` tool declarations are projected into a serializable Pydantic AI toolset request.
- Whether declared tools fail closed with `AA_RUN_TOOL_NOT_FOUND` when no runtime toolset factory is configured.
- Whether function toolsets and MCP toolsets returned by a configured factory are passed to `Agent(toolsets=...)`.
- Whether MCP toolsets are scoped by `server_id` and do not exceed the `NodeContract.mcp_tools` permission upper bound.
- Whether `requires_approval` wraps only the matching MCP server's toolset with Pydantic AI `ApprovalRequiredToolset`.
- Whether `human_in_the_loop`, `deferred_tool_results`, `cancel`, and `evidence_lookup_tool` remain false because SDK resume, cancel, and evidence tool execution are outside W1.4.5.

## First Review

Verdict: FAIL.

Finding:

| ID  | Severity | Description                                                                                                                 | Resolution                                                                                                                        |
| --- | -------- | --------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| F1  | high     | MCP approval lost `server_id` scope and used only `tool_name` / `*`, so same-name tools across servers could be over-gated. | Added `PydanticAIMCPToolset(server_id, toolset)` and wrapped approval per requested server; regression test covers mixed servers. |

## Second Review

Verdict: FAIL.

Finding:

| ID  | Severity | Description                                                                                                                         | Resolution                                                                                                           |
| --- | -------- | ----------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| F2  | high     | Factory-returned but unrequested MCP servers were still passed to `Agent(toolsets=...)`, exceeding the node permission upper bound. | `_sdk_toolsets()` now fails closed with `AA_RUN_TOOL_NOT_FOUND` on unexpected MCP server ids; regression test added. |

## Final Review

Verdict: PASS.

The reviewer confirmed:

- MCP toolsets are bound by `server_id`.
- Missing requested MCP servers fail closed with `AA_RUN_TOOL_NOT_FOUND`.
- Unrequested factory-returned MCP servers fail closed with `AA_RUN_TOOL_NOT_FOUND`.
- Approval wrapping is scoped per requested server.
- `human_in_the_loop`, `deferred_tool_results`, `cancel`, and `evidence_lookup_tool` remain false.
- No production dependency changes were introduced.
- ADR-0002 holds: Pydantic AI remains a lazy import inside the adapter boundary.
